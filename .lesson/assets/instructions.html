Project #1 Part 1 | "Category".csv

Assignment Specification:

Student Generated Category Question Set

  1. Each student must create a category question set between 8 to 20 questions for our Backwards Jeopardy game
    • Our program will ask the questions and the contestant will answer with the answers
    • This is backwards from how the TV show does it where the answers are given and the contestants answers with the question in the form of a question
  2. Each question must:
    • tie to the same category
    • be appropriate for a college course
    • result in a single word ONLY answer
    • CANNOT contain a , (comma) in any part of the category, question or answer as the comma is used to delimit the data Record for each Record
    • assign a score value to each question 100-2000 in increments of 100, where question difficulty is based upon the score
  3. A comma separated file (CSV) named "YourCategory.csv", where "YourCategory" is the name of the category set, and formatted as: Category,Question,Answer,Score
  • The example below would be saved to a file named tennis.csv
  Tennis,How do you represent a score of 0?,Love,100
  Tennis,What is a tennis serve called that hits the net and continues into the correct service box?,let,200
  • Hint: open the .csv file in Excel and if formatted correctly, you'll see 4 columns of data in columns: A,B,C,D. If any data is found in any other columns then you may have included a comma on accident in the question or answer.
    • Commas cannot be part of any of the data itself as it is the delimiter separating the data!

How to Submit:

  1. Name the file YourCategory.csv - where "YourCategory" is the name of the category set.
  2. Upload and submit to the Category.csv to the D2L Assignment Dropbox.

Record Class

  1. Create the Record class that inherits from std::string.
    • Record "is a" std::string

  2. Write constructors in the Record class (that map to std::string constructors), make sure to call the base class constructor and set splitData to nullptr
    1. default (1) | Record() : std::string()
    2. copy (2) | Record(const std::string& str) : std::string(str)
    3. substring (3) | Record(const std::string& str, size_t pos, size_t len=npos) : std::string(str, pos, len)
    4. from c-string (4) | Record(const char* s) : std::string(s)
    5. from sequence (5) | Record(const char* s, size_t n) : std::string(s, n)
    6. fill (6) | Record(size_t n, char c) : std::string(n, c)
  3. Add member functions: upper and lower member functions to convert the Record object to all uppercase or all lowercase, returning the new Record: Record Record::upper();
    • Investigate Python's upper and lower methods to determine structure of the member function and model Record::upper and Record::lower in the same manner
      • https://www.w3schools.com/python/ref_string_upper.asp
      • https://www.w3schools.com/python/ref_string_lower.asp
  4. Add member function: int compareIgnoreCase(const Record& record); in which the function returns 0 if equal, -1 if the first is less than the second and 1 if the first is greater than the second (this function is similiar to std::string::compare(const string& str); member function of string, but is not case sensitive as the std::string::compare is case-sensitive.
    • Investigate C++ std::string::compare member function
      • http://www.cplusplus.com/reference/string/string/compare/
    • The member function uses the *this as the first object and the 2nd object is passed into the function
    • Create a temporary object locally that is assigned *this and convert to upper case
    • Create a temporary object locally that is assigned to the argument and convert to upper case
    • Call the std::string::compare on the 2 temporary objects and return the result
  5. Overload >> for std::istream: std::istream reads until the '\n'
  6. Overload >> for std::ifstream: std::ifstream read until the '\n'
  7. Add member function: Record* split(char delimiter); - this function creates a dynamic array of Record objects based upon the delimiter and populates the array by splitting the Record
    • Investigate Python's split method to determine structure of the member function and task
      • https://www.w3schools.com/python/ref_string_split.asp
    • The split function cannot deallocate the memory; therefore, make a destructor and have the destructor deallocate the memory. The pointer should be a private data member for Record class that is set by the constructors to nullptr and deallocated in the destructor. The split member function allocates the memory, sets the data in the array and returns the pointer out.
  8. Add private data members to assist with the split member function
    • Record* fields;
    • int size, max;
  9. Add public member function to assist with the split member function
    • int getNumFieldsInRecord() const { return size; } // returns number of fields after split
  10. Update all constructors to initialize the new private data members to default values
  11. Implement the destructor to deallocate memory allocated from split - delete [] fields;
    • Destructor Prototype: ~Record();
    • Destructor Definition: Record::~Record() {...}

Node<T> Template Class (Node.h and Node.template)

LinkList<T> Template Class (LinkList.h and LinkList.template)

  1. A LinkList has private data members of

    6. remove by dataItem
    7. removeFirst
    8. clearAll
    9. at(int counter) and returns the item at the location
      • Counter is NOT zero-based nor an index, but a counter of number of items starting at 1

    11. findNode by dataItem return pointer to Node - private member function
    12. bool readFile(std::string fileName); 
    13. bool writeFile(std::string fileName);
    14. Destructor
    15. Copy Constructor
    16. Assignment Operator
  3. Convert LinkList to a template class
    • Update LinkList class (LinkList.h and LinkList.cpp) to a template class and rename LinkList.cpp to LinkList.template

Project #3 Part 1 | BinTree

Objectives: (Show/Hide)

  • Follow and explain tree-based algorithms
  • Design and implement classes for binary tree nodes and binary trees
  • Explain the traversal algorithms for the common binary tree traversals (in-order, pre-order, post-order, backwards in-order)
  • List the rules for a binary search tere and determine whether a tree satisfies these rules
  • Carry out searches, insertions on a binary search tree and implement these algorithms using the binary tree class

Assignment Specificiations:

BinNode<T> Class Template (item in tree data structure)

  1. A BinNode has private data members of
    • BinNode<T>* left
    • BinNode<T>* right
    • T data (data item or element being stored)
  2. The BinNode class needs constructors (default and a constructor that will set the data)
  3. The BinNode class is NOT dynamic
  4. BinNode is a template class that is the helper class to BinTree as the structure of data, left and right pointers.

BinTree<T> Template Class (data structure container of BinNode)

  1. A BinTree is a collection data structure that is dynamic, storing BinNodes.
    • Only BinNode<T>* pointers will be created in BinTree - NO BinNode<T> objects
  2. A BinTree has private data members of
    • BinNode<T>* root
  3. A BinTree has private member functions of
    1. copyTree(BinNode<T>*) - performs preOrder traversal
    2. countNodes(BinNode<T>*, int&) - performs any traversal to count and reassign int&
    3. print(BinNode<T>*, int&) - performs inOrder traversal
    4. writeFile(BinNode<T>*, std::ofstream&) - performs preOrder traversal
    5. free(BinNode<T>*) - performs postOrder traversal
    6. void inOrderRecursive(BinNode<T>* cursor, int& count1, int& count2, void process(T, int&, int&));
      • count1 is NOT incremented and remains the value passed in to the public inOrder member function
      • count2 is incremented up by 1
  4. BinTree is a dynamic template class that needs to ensure full value-semantics is completed
    • Copy Constructor - calls copyTree
    • Assignment Operator - calls copyTree
    • Destructor - calls free
  5. Public Members:
    1. Default Constructor (empty tree) - set root to nullptr
    2. isEmpty - checks if root is nullptr or not
    3. getSize() - returns the number of Nodes - which correlates to the number of data items
    4. add(T dataIn)
    5. readFile(std::string fileName)
    6. printAll() - calls print
    7. writeFile(std::string fileName) - calls private writeFile
    8. void inOrderTraversal(int count1, void process(T, int&, int&)) {
         int count2 = 1;
         inOrderRecursive(root, count1, count2, process);
      }
      • Note that the BinTree::inOrderTraverse and recursive BinTree::inOrderTraverseRecursive match the function call used in the HTMLfunc files (files below) of printHTMLpages
  6. Make sure all the member functions move links NOT data around when adding, removing.
  7. Make sure the class is a template

Bonus BinTree

  • BinTree::remove(T)
  • Setup Friend relationship between BinTree and BinNode so that BinTree can access all private data members of BinNode directly.
